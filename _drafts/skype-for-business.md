Options for running on Linux

The primary issue is that Microsoft does not release a Linux client for Skype for Business. Here is a list of work-arounds I tried:

    I tried using a third party app: https://tel.red/linux.php. This works off and on. I used it for about a month with no problems, but since Thursday it completely stopped working.
    I tried running a Windows VM on VirtualBox. The audio quality was bad as the sound was choppy and people said I sounded like a robot. I tried a number of settings adjustments but the problem did not go away.
    I tried using my Android phone and Android tablet. This worked ok, but it was difficult to see screen shares and impossible to share my own screen. It often would also crash in the middle of a meeting.
    I looked into running SfB on Wine, but the other user reports show that it doesn't work.
    I tried running an Android emulator on my laptop to run the Android version of SfB, but it wouldn't get past the login screen.
    One employee bought a ChromeBook, but SfB would not get past the login screen.
    I believe another employee tried making a Windows GCE container. I don't know the details but I heard it didn't end up working.
    Finally, last night I went out and bought an inexpensive Windows desktop and I have it hooked up without keyboard/mouse/monitor and I have it running NoMachine so that I can remote into it from my Linux Laptop to use SfB and I have it hooked up to my bluetooth headset. WebAdMIT is currently being migrated to SQL Server, so I may be able to use the desktop for that as well. This seems to be the best solution.
